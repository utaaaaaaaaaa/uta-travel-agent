# 数据流设计

## 概述

本文描述 UTA Travel Agent 的数据流设计，包括数据如何在不同组件之间流动。

## 主要数据流

### Agent 创建流程

```
用户输入 ──▶ 前端 ──▶ Orchestrator ──▶ Destination Agent
                                        │
                                        ▼
                    ┌─────────────────────────────────┐
                    │  1. 搜索目的地信息           │
                    │  2. 处理和结构化文档        │
                    │  3. 创建向量嵌入           │
                    │  4. 存储到 Qdrant            │
                    │  5. 元数据存储到 PostgreSQL │
                    └─────────────────────────────────┘
```

### 实时导游流程

```
用户位置/图片 ──▶ 前端 ──▶ Orchestrator ──▶ Guide Agent
                                        │
                                        ▼
                    ┌─────────────────────────────────┐
                    │  1. 识别位置/图片           │
                    │  2. 查询 Qdrant 向量库      │
                    │  3. 使用 Claude API 生成讲解 │
                    │  4. 返回给用户              │
                    └─────────────────────────────────┘
```

## 数据存储

| 存储系统 | 用途 | 数据类型 |
|--------|------|--------|
| PostgreSQL | Agent 元数据 | 结构化数据 |
| Qdrant | 向量嵌入 | 向量数据 |
| Redis | 会话缓存 | 临时数据 |
| MinIO | 文件存储 | 图片、文档 |

## API 设计

- **REST API**: 前端与 Orchestrator 通信
- **gRPC**: Orchestrator 与 Agent Services 通信
- **WebSocket**: 实时导游通信

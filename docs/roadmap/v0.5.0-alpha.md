#### 1.2 改造 Agent 创建流程
```go
// router.go - executeAgentCreation
func (r *Router) executeAgentCreation(task *agent.AgentTask, ag *agent.DestinationAgent) {
    // 不再使用 simulateProgress

    // 1. 创建 LLMAgent (Researcher)
    researcher := agent.NewLLMAgent(agent.LLMAgentConfig{
        AgentType:   agent.AgentTypeResearcher,
        LLMProvider: r.llmProvider,
        Tools:       []Tool{webSearchTool, webReaderTool},
    })

    // 2. 执行研究任务
    result, err := researcher.Run(ctx, fmt.Sprintf("收集 %s 的旅游信息", ag.Destination))

    // 3. 创建向量索引
    collectionID := r.createVectorIndex(ctx, result.Documents)

    // 4. 更新 Agent 状态
    ag.VectorCollectionID = collectionID
    ag.Status = agent.StatusReady
}
```

**任务清单**:
- [ ] 移除 simulateProgress
- [ ] 接入真实 LLMAgent
- [ ] 实现向量索引创建

## 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 搜索 API | SerpAPI | 成熟稳定，有免费额度 |
| LLM 流式 | SSE | 已有实现，前端兼容性好 |
| Subagent | LLMAgent | 已有 ReAct 循环实现 |
| 会话缓存 | Redis (可选) | 提升响应速度 |

## 风险和缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 搜索 API 限制 | Agent 创建失败 | 添加缓存，支持多 Provider |
| LLM 响应慢 | 用户体验差 | 流式输出 + 缓存 |
| 向量索引失败 | 数据丢失 | 添加重试 + 本地备份 |
| Token 消耗大 | 成本高 | 优化 Prompt，添加缓存 |
